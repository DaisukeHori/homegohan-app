# 国際化 (i18n) / アクセシビリティ (a11y) 設計

## 1. 目的・スコープ

全ドメイン共通の国際化対応と WCAG 2.1 AA 準拠のアクセシビリティ設計を定義する。

### 3.4 next-intl セットアップ (Web)

```typescript
// src/i18n/routing.ts
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['ja'],  // Phase 2 で ['ja', 'en'] に拡張
  defaultLocale: 'ja',
  localePrefix: 'never',  // URL パスに /ja/ を含めない
});
```

```typescript
// src/i18n/request.ts
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  const locale = (await requestLocale) ?? routing.defaultLocale;
  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default,
  };
});
```

```tsx
// src/app/layout.tsx
import { NextIntlClientProvider } from 'next-intl';
import { getMessages } from 'next-intl/server';

export default async function RootLayout({ children }: { children: React.ReactNode }) {
  const messages = await getMessages();
  return (
    <html lang="ja">  {/* Phase 2: lang を動的に */}
      <body>
        <NextIntlClientProvider messages={messages}>
          {children}
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

## 4. アクセシビリティ (a11y)

### 4.1 ターゲット水準

**WCAG 2.1 Level AA** 準拠 (法的根拠: 米国 ADA / EU EAA / 日本 JIS X 8341-3:2016)

### 4.2 セマンティック HTML

```tsx
// ✅ 正しい実装
<button onClick={handleDelete} type="button">削除</button>
<a href="/family/groups">家族グループ</a>
<form onSubmit={handleSubmit}>...</form>
<nav aria-label="メインナビゲーション">...</nav>
<main id="main">...</main>  {/* Skip to main content のターゲット */}

// ❌ 禁止
<div onClick={handleDelete}>削除</div>  {/* div のクリックハンドラ */}
<span role="button">送信</span>         {/* button を使うべき */}
```

### 4.3 ARIA 属性規約

#### 必須 ARIA

```tsx
// アイコンのみのボタン
<button aria-label="家族グループを削除">
  <Trash2 size={16} aria-hidden="true" />
</button>

// フォームエラー
<input
  id="email"
  aria-describedby="email-error"
  aria-invalid={!!errors.email}
/>
<p id="email-error" role="alert">{errors.email?.message}</p>

// 動的更新 (献立提案完了等)
<div aria-live="polite" aria-atomic="true">
  {status === 'completed' && '献立提案が完了しました'}
</div>

// 展開/折りたたみ
<button
  aria-expanded={isOpen}
  aria-controls="panel-id"
>
  詳細を表示
</button>
<div id="panel-id" hidden={!isOpen}>...</div>

// 選択状態
<li role="option" aria-selected={isSelected}>...</li>

// モーダル
<div
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-title"
  aria-describedby="modal-desc"
>...</div>
```

#### Mobile (React Native)

```tsx
// accessibilityLabel
<TouchableOpacity accessibilityLabel="家族グループを削除" onPress={handleDelete}>
  <Icon name="trash" />
</TouchableOpacity>

// accessibilityHint (操作の説明)
<TextInput
  accessibilityLabel="メールアドレス"
  accessibilityHint="招待メールを送信するメールアドレスを入力してください"
  accessibilityRequired={true}
/>

// accessibilityRole
<Text accessibilityRole="header">家族グループ設定</Text>
<View accessibilityRole="list">...</View>
```

### 4.4 カラーコントラスト

要件: 通常テキスト 4.5:1 以上 / 大テキスト (18pt+) 3:1 以上

検証ツール: Storybook + `@storybook/addon-a11y` / Figma Colour Contrast プラグイン

```typescript
// scripts/check-contrast.ts (CI で実行)
import { meetsContrastGuidelines } from 'polished';

const pairs = [
  { fg: '#111827', bg: '#FAF9F7', label: 'text on bg' },
  { fg: '#6B7280', bg: '#FAF9F7', label: 'text-secondary on bg' },
  { fg: '#FFFFFF', bg: '#E07A5F', label: 'white on primary' },
  { fg: '#EF4444', bg: '#FFFFFF', label: 'danger on white' },
];

pairs.forEach(({ fg, bg, label }) => {
  const result = meetsContrastGuidelines(fg, bg);
  if (!result.AA) {
    console.error(`FAIL: ${label} does not meet AA (${fg} on ${bg})`);
    process.exit(1);
  }
});
```

### 4.5 キーボードナビゲーション

```tsx
// Skip to main content (全ページ先頭)
export function SkipLink() {
  return (
    <a
      href="#main"
      className="sr-only focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-50 focus:px-4 focus:py-2 focus:bg-primary focus:text-white focus:rounded-md"
    >
      メインコンテンツへスキップ
    </a>
  );
}
```

**フォーカス管理規約**:
- モーダル open 時: 最初のフォーカス可能要素 (または `DialogTitle`) に `focus()`
- モーダル close 時: トリガーボタンに `focus()` を戻す
- フォーム submit エラー時: 最初のエラーフィールドに `focus()`
- `outline: none` のみの設定禁止 (フォーカスリングの非表示禁止)

```tsx
// フォーカストラップ (モーダル内)
import { FocusTrap } from 'focus-trap-react';

export function Modal({ open, onClose, children }: ModalProps) {
  return open ? (
    <FocusTrap
      focusTrapOptions={{
        onDeactivate: onClose,
        clickOutsideDeactivates: true,
        returnFocusOnDeactivate: true,
      }}
    >
      <div role="dialog" aria-modal="true">
        {children}
      </div>
    </FocusTrap>
  ) : null;
}
```

**キーボード操作一覧**:

| キー | 動作 |
|------|------|
| `Tab` | 次のフォーカス可能要素へ |
| `Shift + Tab` | 前のフォーカス可能要素へ |
| `Enter` | ボタン実行 / リンク遷移 |
| `Space` | チェックボックス toggle / ボタン実行 |
| `Esc` | モーダル閉じる / ドロップダウン閉じる |
| `↑↓` | リスト / ドロップダウン内のナビゲーション |
| `Home` / `End` | リスト先頭 / 末尾へ |

### 4.7 prefers-reduced-motion

詳細は cross/03-design-system.md §14.2 を参照。

Mobile での対応:
```tsx
// apps/mobile/src/lib/accessibility.ts
import { AccessibilityInfo } from 'react-native';

export function useReducedMotion(): boolean {
  const [reducedMotion, setReducedMotion] = React.useState(false);
  React.useEffect(() => {
    AccessibilityInfo.isReduceMotionEnabled().then(setReducedMotion);
    const listener = AccessibilityInfo.addEventListener(
      'reduceMotionChanged',
      setReducedMotion
    );
    return () => listener.remove();
  }, []);
  return reducedMotion;
}
```

## 9. ハンズオンチュートリアル i18n / a11y (family/09 連携)

family/09 初回オンボーディングハンズオンチュートリアルの i18n key 設計と a11y 要件の canonical。詳細リスト・コードサンプルは family/09 §14-mocks-i18n.md / §10-a11y.md 参照。

### 9.1 i18n キー設計

ja v1 のみ。配置は workspace package `packages/handson-tour-shared/src/i18n.ts`。Web/Mobile 両方が import。

#### 9.1.1 キー命名規則

- ケース: snake_case
- 全 81 キー(セクション別件数: step0=8 / step1=14 / step2=14 / step3=13 / step4=12 / step5=2 / cooking_experience=3 / a11y=9 / common=6)

#### 9.1.2 確定文言(§99 §1.2 確定)

- `tour.step0.title` = `'{nickname} さん、ようこそ!'`
- `tour.step0.subtitle` = `'3 つの便利機能を一緒に試してみましょう (約 90 秒)'` (§99 §1.2 Q2)

#### 9.1.5 v1 段階での言語

ja のみ提供。en は v2 で追加(§99 Q24 open、`packages/handson-tour-shared/src/i18n.en.ts` に mirror 配置予定)。本セクションが canonical のため en 追加時は本ファイルにも反映する。

### 9.2 アクセシビリティ要件

§4(a11y セクション)の規約をハンズオンチュートリアルに適用した最小要件。Coachmark コンポーネントの ARIA 仕様は cross/03 §21.7 と重複しないよう、本セクションは **画面・動作レベル** の要件に絞る。

#### 9.2.1 セマンティック要素 (§4.2 連携)

- ハンズオン Tour 全体は `<section role="region" aria-labelledby="tour-title">` でくくる
- `tour-title` には Step ごとの `tour.stepN.a11y_title` を充てる(全 Step でユニーク)
- skip ボタンは `<button>` 要素で実装(`<a>` でリンク偽装しない)

#### 9.2.2 ARIA ライブリージョン (§4.3 連携)

- TourBubble: `role="status"` + `aria-live="polite"`(吹き出し更新を読み上げ)
- Step 1 結果カード: `tour.step1.a11y_result_announce` を `aria-live="polite"` で発火
- Step 4 卒業: `tour.step4.a11y_announce` を発火(紙吹雪は `aria-hidden="true"`)
- Step 5 welcome toast: `tour.step5.a11y_toast_announce` を発火(toast 4 秒後に消えるが、読み上げ完了まで待機)

#### 9.2.3 キーボードナビゲーション (§4.5 連携)

- Web: Tab で `primary action` → `skip` へ移動、Esc で skip 発火、Space/Enter で primary action
- focus trap: `<TourOverlay>` 内に focus を閉じ込める(`focus-trap-react` を使用、§9 既存未解決事項の互換確認は実装時に対応)
- Mobile: native focus order、accessibility actions で skip 発火

#### 9.2.4 スクリーンリーダー (§4.6 連携)

- VoiceOver / TalkBack で各 Step のタイトル → 本文 → primary action の順に読み上げ
- 進捗ドット個別は `aria-hidden="true"`、`<TourProgress>` 全体に `aria-valuenow` / `aria-valuetext="ステップ {current} / {total}"` を付与
- 個別ドット読み上げの冗長性は §99 Q18 open(Phase 4 で aria-hidden vs aria-valuetext 単発の切替判断)

#### 9.2.5 prefers-reduced-motion (§4.7 連携)

- entrance/exit fade の duration を 0 に置換
- 紙吹雪(Step 4)は `prefers-reduced-motion: reduce` 時に表示しない、代わりに静的アイコン(🎓)のみ
- Mobile は `AccessibilityInfo.isReduceMotionEnabled()` を購読し同等動作

#### 9.2.6 カラーコントラスト (§4.4 連携)

- 吹き出し背景(white)に本文(`--color-text-secondary`):4.5:1 以上
- primary action(`--color-primary-600` 背景に white テキスト):4.5:1 以上
- dim 背景(opacity 0.6)に skip ボタン(`white/80`):3:1 以上(大型テキスト基準)
- iOS Dynamic Type AX5(2.85x)で吹き出しはみ出ないこと → §99 Q17 open、Phase 4 で実機検証

### 9.3 文言マスタ更新の凍結ポリシー

mock データ(`MOCK_PHOTO_RESPONSE` の `'鶏の唐揚げ定食'` 等)の値は v1 確定後 freeze。理由: i18n キーの一部に値が埋め込まれている(例: `'鶏の唐揚げ定食、780 kcal の約 X%'`)。変更は v2 で実施。

### 9.4 数値定数の統一

タイマー / アニメーション / タイムアウトの全数値は `packages/handson-tour-shared/src/constants.ts` の `HANDSON_TOUR_CONSTANTS` に集約(family/09 §14 §5.1 参照)。マジックナンバーは禁止。

主要値:
- 各ステップ自動進行: 2000〜2500ms
- アナライザー擬似 loading: 1500ms
- 卒業画面ボタン活性化: 5000ms
- welcome toast 表示: 4000ms
- overlay フェード: 200ms / spotlight 移動: 250ms / bubble フェード: 150ms
- 紙吹雪パーティクル数: 300 / 表示時間: 3000ms
