# 国際化 (i18n) / アクセシビリティ (a11y) 設計

## 1. 目的・スコープ

全ドメイン共通の国際化対応と WCAG 2.1 AA 準拠のアクセシビリティ設計を定義する。  
各ドメイン実装はここで定めたライブラリ・規約・ツールを使用する。

**対象外**: ドメイン固有の UI テキスト内容 (各ドメインの ui-spec.md で管理)

## 3. 国際化 (i18n)

### 3.1 採用ライブラリ

| 環境 | ライブラリ | バージョン |
|------|---------|----------|
| Web (Next.js) | `next-intl` | ^3.x |
| Mobile (Expo) | `i18next` + `react-i18next` | ^23.x |

### 3.2 フェーズ別対応言語

| フェーズ | 言語 |
|---------|------|
| Phase 1 | 日本語のみ |
| Phase 2 | 日本語 + 英語 (org_enterprise 顧客要望次第) |

プラン別提供言語:
- `free` / `pro` / `family_*`: 日本語のみ
- `org_*` (Phase 2 以降): 日本語 + 英語

### 3.3 リソースファイル配置

```

├── ja.json          # Web: 日本語 (Phase 1 〜)
└── en.json          # Web: 英語 (Phase 2 〜)

apps/mobile/locales/
├── ja.json          # Mobile: 日本語 (Phase 1 〜)
└── en.json          # Mobile: 英語 (Phase 2 〜)
```

### 3.4 next-intl セットアップ (Web)

```typescript
// src/i18n/routing.ts
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['ja'],  // Phase 2 で ['ja', 'en'] に拡張
  defaultLocale: 'ja',
  localePrefix: 'never',  // URL パスに /ja/ を含めない
});
```

```typescript
// src/i18n/request.ts
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  const locale = (await requestLocale) ?? routing.defaultLocale;
  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default,
  };
});
```

```tsx
// src/app/layout.tsx
import { NextIntlClientProvider } from 'next-intl';
import { getMessages } from 'next-intl/server';

export default async function RootLayout({ children }: { children: React.ReactNode }) {
  const messages = await getMessages();
  return (
    <html lang="ja">  {/* Phase 2: lang を動的に */}
      <body>
        <NextIntlClientProvider messages={messages}>
          {children}
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

**キー命名規則**:
- ドット区切りの階層構造
- camelCase (JSON キー)
- ドメイン名 → エンティティ → アクション / 状態 の順
- エラーコードはそのままキーとして使用 (`error.AUTH_UNAUTHENTICATED`)

### 3.6 使用パターン

```tsx
// Server Component (推奨)
import { getTranslations } from 'next-intl/server';

export async function FamilyGroupCard() {
  const t = await getTranslations('family.group');
  return <h2>{t('create')}</h2>;
}

// Client Component
'use client';
import { useTranslations } from 'next-intl';

export function ErrorMessage({ code }: { code: string }) {
  const t = useTranslations('error');
  return <p className="text-danger">{t(code, { fallback: t('default') })}</p>;
}
```

## 4. アクセシビリティ (a11y)

### 4.1 ターゲット水準

**WCAG 2.1 Level AA** 準拠 (法的根拠: 米国 ADA / EU EAA / 日本 JIS X 8341-3:2016)

### 4.2 セマンティック HTML

```tsx
// ✅ 正しい実装
<button onClick={handleDelete} type="button">削除</button>
<a href="/family/groups">家族グループ</a>
<form onSubmit={handleSubmit}>...</form>
<nav aria-label="メインナビゲーション">...</nav>
<main id="main">...</main>  {/* Skip to main content のターゲット */}

// ❌ 禁止
<div onClick={handleDelete}>削除</div>  {/* div のクリックハンドラ */}
<span role="button">送信</span>         {/* button を使うべき */}
```

### 4.3 ARIA 属性規約

#### 必須 ARIA

```tsx
// アイコンのみのボタン
<button aria-label="家族グループを削除">
  <Trash2 size={16} aria-hidden="true" />
</button>

// フォームエラー
<input
  id="email"
  aria-describedby="email-error"
  aria-invalid={!!errors.email}
/>
<p id="email-error" role="alert">{errors.email?.message}</p>

// 動的更新 (献立提案完了等)
<div aria-live="polite" aria-atomic="true">
  {status === 'completed' && '献立提案が完了しました'}
</div>

// 展開/折りたたみ
<button
  aria-expanded={isOpen}
  aria-controls="panel-id"
>
  詳細を表示
</button>
<div id="panel-id" hidden={!isOpen}>...</div>

// 選択状態
<li role="option" aria-selected={isSelected}>...</li>

// モーダル
<div
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-title"
  aria-describedby="modal-desc"
>...</div>
```

#### Mobile (React Native)

```tsx
// accessibilityLabel
<TouchableOpacity accessibilityLabel="家族グループを削除" onPress={handleDelete}>
  <Icon name="trash" />
</TouchableOpacity>

// accessibilityHint (操作の説明)
<TextInput
  accessibilityLabel="メールアドレス"
  accessibilityHint="招待メールを送信するメールアドレスを入力してください"
  accessibilityRequired={true}
/>

// accessibilityRole
<Text accessibilityRole="header">家族グループ設定</Text>
<View accessibilityRole="list">...</View>
```

### 4.4 カラーコントラスト

要件: 通常テキスト 4.5:1 以上 / 大テキスト (18pt+) 3:1 以上

検証ツール: Storybook + `@storybook/addon-a11y` / Figma Colour Contrast プラグイン

```typescript
// scripts/check-contrast.ts (CI で実行)
import { meetsContrastGuidelines } from 'polished';

const pairs = [
  { fg: '#111827', bg: '#FAF9F7', label: 'text on bg' },
  { fg: '#6B7280', bg: '#FAF9F7', label: 'text-secondary on bg' },
  { fg: '#FFFFFF', bg: '#E07A5F', label: 'white on primary' },
  { fg: '#EF4444', bg: '#FFFFFF', label: 'danger on white' },
];

pairs.forEach(({ fg, bg, label }) => {
  const result = meetsContrastGuidelines(fg, bg);
  if (!result.AA) {
    console.error(`FAIL: ${label} does not meet AA (${fg} on ${bg})`);
    process.exit(1);
  }
});
```

### 4.5 キーボードナビゲーション

```tsx
// Skip to main content (全ページ先頭)
export function SkipLink() {
  return (
    <a
      href="#main"
      className="sr-only focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-50 focus:px-4 focus:py-2 focus:bg-primary focus:text-white focus:rounded-md"
    >
      メインコンテンツへスキップ
    </a>
  );
}
```

**フォーカス管理規約**:
- モーダル open 時: 最初のフォーカス可能要素 (または `DialogTitle`) に `focus()`
- モーダル close 時: トリガーボタンに `focus()` を戻す
- フォーム submit エラー時: 最初のエラーフィールドに `focus()`
- `outline: none` のみの設定禁止 (フォーカスリングの非表示禁止)

```tsx
// フォーカストラップ (モーダル内)
import { FocusTrap } from 'focus-trap-react';

export function Modal({ open, onClose, children }: ModalProps) {
  return open ? (
    <FocusTrap
      focusTrapOptions={{
        onDeactivate: onClose,
        clickOutsideDeactivates: true,
        returnFocusOnDeactivate: true,
      }}
    >
      <div role="dialog" aria-modal="true">
        {children}
      </div>
    </FocusTrap>
  ) : null;
}
```

**キーボード操作一覧**:

| キー | 動作 |
|------|------|
| `Tab` | 次のフォーカス可能要素へ |
| `Shift + Tab` | 前のフォーカス可能要素へ |
| `Enter` | ボタン実行 / リンク遷移 |
| `Space` | チェックボックス toggle / ボタン実行 |
| `Esc` | モーダル閉じる / ドロップダウン閉じる |
| `↑↓` | リスト / ドロップダウン内のナビゲーション |
| `Home` / `End` | リスト先頭 / 末尾へ |

### 4.6 スクリーンリーダー対応

**対象スクリーンリーダー**:
- iOS: VoiceOver
- Android: TalkBack
- Web: NVDA (Windows) / VoiceOver (macOS)

**テスト必須フロー**:
1. ログイン → ダッシュボード到達
2. 家族グループ作成
3. 献立リクエスト送信
4. フォームエラー確認
5. モーダル操作 (開閉 + 実行)

**実装規約**:
```tsx
// ローディング状態の通知
<div role="status" aria-live="polite" aria-label="読み込み状態">
  {isLoading ? '読み込み中です' : ''}
</div>

// ページ遷移後のアナウンス (SPA)
<div role="alert" aria-live="assertive" className="sr-only">
  {pageTitle && `${pageTitle} に移動しました`}
</div>

// 件数のアナウンス
<p aria-live="polite" className="sr-only">
  {`${results.length} 件見つかりました`}
</p>
```

### 4.7 prefers-reduced-motion

詳細は cross/03-design-system.md §14.2 を参照。

Mobile での対応:
```tsx
// apps/mobile/src/lib/accessibility.ts
import { AccessibilityInfo } from 'react-native';

export function useReducedMotion(): boolean {
  const [reducedMotion, setReducedMotion] = React.useState(false);
  React.useEffect(() => {
    AccessibilityInfo.isReduceMotionEnabled().then(setReducedMotion);
    const listener = AccessibilityInfo.addEventListener(
      'reduceMotionChanged',
      setReducedMotion
    );
    return () => listener.remove();
  }, []);
  return reducedMotion;
}
```

### 5.2 axe-core/playwright 統合

```typescript
// tests/e2e/a11y.spec.ts
import { test, expect } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

const pages = [
  { name: 'ログイン', url: '/login' },
  { name: 'ダッシュボード', url: '/dashboard' },
  { name: '家族グループ', url: '/family/groups' },
  { name: '組織設定', url: '/org/settings' },
];

for (const { name, url } of pages) {
  test(`a11y violations = 0 on ${name}`, async ({ page }) => {
    await page.goto(url);
    const results = await new AxeBuilder({ page })
      .withTags(['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa'])
      .analyze();
    expect(results.violations).toEqual([]);
  });
}
```

## 7. テスト方針

| テスト種別 | 対象 | ツール |
|---------|------|------|
| Lint | ハードコード文字列検出、ARIA 属性欠如 | ESLint (CI 必須) |
| a11y | 全主要画面で violations = 0 | @axe-core/playwright (CI 必須) |
| Visual | カラーコントラスト検証 | contrast-check スクリプト |
| 手動 | VoiceOver / TalkBack での主要フロー操作 | 各リリース前 |
| Unit | `useReducedMotion`, i18n キー存在確認 | Vitest |
